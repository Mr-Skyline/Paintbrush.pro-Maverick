Build a standalone AI takeoff agent that processes construction blueprints (PDFs/images) autonomously—no On-Screen Takeoff or external tools. It detects walls as lines, rooms as polygons, doors/windows as counts/symbols, calculates quantities (EA/SF/LF), exports JSON/CSV for Supabase handoff. Use Python 3.12+, OpenCV, YOLOv8/v11, PaddleOCR, Torch for CV/ML. Make it modular, trainable, accurate >90% on clean plans like Kirksey floors.

2. Input Handling

Accept: PDF, PNG/JPG, multi-page.
Convert PDF → high-res images (300 DPI) via pdf2image.
Auto-detect scale: OCR for "1/8"=1'" text, or assume default; user override via config.
Denoise: Gaussian blur + adaptive threshold (OpenCV).

3. Core Detection Pipeline (inspired by Togal + Floor-Plan-Detection)

Wall Detection (lines/segments):
Use YOLOv11 trained on floor plan walls (CubiCasa5k dataset or custom).
Output: bounding boxes + keypoints for endpoints.
Merge overlapping lines → skeletonize (OpenCV morphology) → vector paths.
Classify types: red=exterior, blue=interior via color histogram.

Area Extraction (polygons/rooms):
Segmentation model (U-Net or SAM2) on walls → room masks.
Contour detection (OpenCV findContours) → approximate polygons (cv2.approxPolyDP).
Close shapes: snap angles to 90°/45°, expand by 4" for tolerances (like Togal).
Calculate SF: shapely Polygon.area * scale factor.
Ignore <0'3" features (filter noise).

Counts (doors/windows/fixtures):
YOLO for object detection (doors as symbols).
Keypoints: hinge/open/close points → orientation (virtual mirror method).
Count EA: cluster by type (Levenshtein on OCR labels), center-click sim.
Chain bonus: same-type groups get +points in training.


4. Accuracy & Training

Use CubiCasa5k or custom dataset (label 500+ plans: walls, rooms, doors).
Fine-tune: YOLO for detection, SAM for segmentation.
Metrics: mAP@0.5 >0.9, polygon IoU >0.92, count accuracy >95%.
Reward shaping: +12 if area >92% match, -4 if <85%; snap to grid.
Qualification: simulate button centers—must hit 85% before full run.
